親ラズパイの起動と
2つの無線の役割

電源を入れてから記録が始まるまで。内蔵WiFiとUSBドングル、それぞれ何をしていて、どう使い分ければいいの？

所要 6分 / 図解多め・中学生でも読めるように
①

親ラズパイって何をする子？

親ラズパイは 「工場のOracleにデータをあげる代表係（ハブ）」。子ラズパイたちのデータを集め、まとめてOracleに書き込みます。

そのために親は無線を2枚持っています。1枚は内蔵WiFi、もう1枚は後付けのUSBドングル。この2枚を上手に使い分けるのが運用のキモです。

🏢 会社の受付＆経理に似ています。受付（ドングル）は来客（子ラズパイ）を迎え入れ、経理（内蔵WiFi）は本社の金庫（Oracle）とやりとりする。窓口が2つあるから、来客対応と本社連絡を同時にこなせます。
🍓 親ラズパイ（ハブ）
Raspberry Pi 5 / 4コンテナが常駐
📶 内蔵WiFi wlan0
driver: brcmfmac
許可されたMACを持つ「正規回線」
↓
工場網 → Oracle
🔌 ドングル wlan1
driver: rtl8821au (USB)
用途を切り替える「増設の窓口」
↓
子Piの親機(AP) or ネット
②

起動のしかた（電源だけ）

難しい操作は不要。電源を入れれば必要なプログラムは自動で立ち上がります。ただし「工場につなぐ」のは手動です。

電源を入れる

USB-C電源をつなぐだけ。OSが起動する。

4つのコンテナが自動起動

systemd と Docker の自動再起動で、記録に必要な4つの部品が勝手に立ち上がる。

docker ps で4つ Up になる

この時点では「工場未接続」

電源ONだけでは工場網につながらない（安全のため自動接続しない）。まだOracleへはあげていないがOK。

運用モードを選ぶ

本番にするなら Desktop の「HIME-H-REAP 接続」を押す（⑤章）。これで工場とつながり記録開始。

自動で立ち上がる4つの部品（コンテナ）：

📬 受け口

mosquitto

MQTTの郵便受け。子Piや検知の結果を受け取る。

👁 検知

detector

このPiのカメラで在席を判定（カメラがあれば）。

🧮 転記

bridge

受けたデータをOracleへ書き込む司令塔。

🔌 橋渡し

oracle-jdbc

Oracleとつなぐ専用サイドカー（Java）。

ヒント💡 再起動しても4つは自動で復活します。だから「電源を入れ直す」だけで土台は元通り。あとは工場につなぐ操作（アイコン）をするだけ。
③

2つの無線、それぞれの役割

無線は2枚。内蔵(wlan0)は工場専用、ドングル(wlan1)は用途を切り替える窓口、と覚えると迷いません。

📶
内蔵WiFi

wlan0

工場のOT網「HIME-H-REAP」につなぐ専用。許可されたMACはこの内蔵側だけなので、工場接続は必ずここ。Oracle/NTPへの通り道。

brcmfmac / 工場サブネットのみ・既定経路は奪わない
🔌
USBドングル

wlan1

役割を切り替える窓口。ふだんは子Piの親機(AP)「presence-hub」。開発時はインターネット(UFI)受けにも使える。

rtl8821au / AP=10.42.0.1（子Piは10.42.0.x）
なぜ固定？⚠️ 工場網はMACアドレス許可制。登録されているのは内蔵(wlan0)のMACだけなので、工場接続を内蔵に固定します。番号(wlan0/wlan1)ではなく「内蔵＝工場／ドングル＝子Pi」で覚えてください。

ドングルがAPのとき、子Piは1:多でぶら下がれます：

🍓 親ラズパイ wlan1=AP
SSID: presence-hub / 10.42.0.1
🍓 子Pi-01
10.42.0.x
MQTTで投函
🍓 子Pi-02
10.42.0.x
同じ宛先へ
🍓 子Pi-03
10.42.0.x
何台でもOK
④

3つの運用モード

無線は2枚しかないので、3つの役割（工場・ネット・子PiAP）のうち同時に持てるのは2つ。目的でモードを選びます。

🏭
本番モード

工場 ＋ 子PiAP

記録の本番。wlan0=工場(Oracle) / wlan1=AP(子Pi)。インターネットは無し。

Desktop「HIME-H-REAP 接続」アイコン
🛠️
開発モード

ネット ＋ 子PiAP

開発・動作確認用。wlan0=UFI(ネット) / wlan1=AP(子Pi)。ネットありで子Piテストもできる。

sudo bash mode-dev.sh
🔬
検証モード

工場 ＋ ネット

Oracle書込みを見ながら確認したいとき。wlan0=工場 / wlan1=UFI(ネット)。子PiAPは無し。

dual-WiFi（手動 nmcli）
大事⚠️ 本番モードはインターネットが消えます（工場OT網は外に出られない）。工場外からSSH等で管理している場合、本番接続中は切れます。管理に戻すには「切断」でネット側へ。
⑤

毎日の操作（アイコン2つ）

ふだんはDesktopの2つのアイコンだけ。「接続」で始め、「切断」で止める。

記録を始めたい
→
Desktop HIME-H-REAP 接続
記録を止めたい
→
Desktop HIME-H-REAP 切断
入った記録を見たい
→
Desktop 直近30件の記録

「接続」を押すと…

wlan0=工場 を張り、時刻同期し、wlan1=AP を起動、子Pi用にMQTTを公開、検知も開始。これ一発で本番状態になる。

工場・子PiAP・記録が一括ON

記録が流れる

このPiのカメラ＋子Piからの送信が、bridge経由でOracle(HHC001)へ。通信が一時切れても貯めて後で送る。

「切断」を押すと…

検知を止め、子PiAPを止め、工場網を切り、ネット(UFI)へ戻す。両方の通信がオフになり、管理に戻れる。

工場・子PiAP・検知が一括OFF
ヒント💡 隠しSSID＋5GHzの工場APは、つながるまで少し時間がかかることがあります。「接続」直後にすぐ反応しなくても、数十秒待てばつながります（待ち時間は調整済み）。
⑥

確認のしかたと注意点

「ちゃんと記録できている？」は、見るところが決まっています。

✓ 確認

直近30件の記録

Oracleを直接見て、最新の記録が入っているか確かめられる（Desktopアイコン）。

✓ 確認

時刻が合っているか

工場NTPで自動同期。合っていれば工場接続が健全な目安にもなる。

⚠ 注意

本番中はネット無し

本番モードはインターネット不可。リモート管理は「切断」後に。

⚠ 注意

ドングルは抜かない

子PiAPはドングル。抜くと子Piの受け口が消える（工場記録自体は内蔵で継続）。

✓

迷ったら「内蔵=工場／ドングル=子Pi」

wlan0/wlan1の番号は起動順で入れ替わり得る。役割はドライバ(brcmfmac/rtl8821au)で見分ける。

!

アイコンの実体は ~/Desktop 側

スクリプトを直しても、アイコンが動かすのは ~/Desktop/presence-tools のコピー。更新時は同期が必要。

ひとことで言うと

親ラズパイは 電源ONで土台が自動起動。あとは「接続」で本番（工場＋子PiAP）、「切断」で停止。無線は内蔵=工場、ドングル=子Pi。

  • 起動：電源を入れるだけ → 4コンテナが自動起動（工場接続は手動）
  • 内蔵 wlan0：工場網 HIME-H-REAP → Oracle（許可MACはここだけ）
  • ドングル wlan1：子Piの親機(AP) presence-hub（開発時はネット受けにも）
  • モード：本番=工場+AP／開発=ネット+AP／検証=工場+ネット（同時は2つまで）
  • 操作：Desktop「接続」で開始・「切断」で停止・「直近30件」で確認
